# Daily Loop-Model Briefing — 2026-06-01

## Outcome

One paper was added to the collection. It is a theory-oriented analysis paper about the memory limits of compressed looped Transformers compared with chain-of-thought scratchpads.

## Added to the collection

### Chain-of-Thought and Compressed Looped Transformers: A Memory-Budget Separation

This paper compares two ways a fixed model can spend extra test-time computation: external chain-of-thought tokens and internal looped Transformer recurrence. Its core claim is that these mechanisms differ in what they remember. Chain-of-thought can grow a token scratchpad, while a compressed looped Transformer carries only a bounded recurrent hidden state.

Why it was added: the paper is directly about looped Transformers and analyzes recurrent hidden activations as the model's persistent mutable memory. It belongs in analysis because the main contribution is a memory-budget separation, not a new looped architecture.

Main reader takeaway: running a compressed loop for more steps adds computation, but does not by itself create an expanding scratchpad. The paper argues that this matters for reasoning tasks whose working-memory demand exceeds the loop's persistent state budget.
