# Daily Loop-Model Briefing - 2026-06-23

## Outcome

One paper was added to the collection. It is not a language-model looped Transformer, but it fits the same strict computational pattern: a learned recurrent update is applied repeatedly inside one model process, and the number of recurrence steps becomes a test-time compute knob.

## Added to the collection

### Adaptive Recurrent Message Passing for Test Time Computing on Graphs

AdaR targets graph learning, where fixed-depth architectures can be too rigid across different graph tasks. The paper derives step dependence as the condition needed for adaptive convergence, then builds a recurrent graph model whose updates encode normalized step information and representation-target relations.

Why it was added: the central mechanism is recurrent learned message passing. AdaR runs repeated internal representation updates without changing model parameters, making recurrence depth a flexible test-time compute axis rather than an external agent loop or repeated full-model call.

Main reader takeaway: this is a graph-domain example of loop-model compute scaling. The interesting part for this collection is the explicit recurrent update design and convergence supervision, not just better graph benchmark numbers.

## Checked but not added

- **Reasoning as Attractor Dynamics: Latent Memory Retrieval via Gibbs-Weighted Energy Minimization** (`2606.24543`) frames LLM reasoning as settling into attractor basins, but the operational mechanism samples multiple output reasoning paths and reweights them by an energy score. That is adjacent inference-time aggregation, not a reused learned internal module.
- **Compressing Observation History into Agent Memory: Distilling Transformers into Recurrent Transformers** (`2606.21562`) is useful recurrent-memory work for streaming vision and robotics, but the abstract emphasizes sequence-memory compression over time rather than recurrent-depth reuse within one forward process.
- **Compression and Retrieval: Implicit Memory Retrieval for Video World Models** (`2606.23105`) adds attention-driven implicit memory retrieval plus context compression for video world models. Retrieval over compressed context is not enough by itself to meet the strict loop-model criterion.
- **IV-CoT: Implicit Visual Chain-of-Thought for Structure-Aware Text-to-Image Generation** (`2606.24849`) performs latent visual planning in a single forward pass, but the abstract does not identify a reused internal learned block or fixed-point computation.
