# Daily Loop-Model Briefing - 2026-06-25

## Outcome

One new paper was added to the collection. It is not a new looped architecture proposal, but it directly studies what happens when recursive reasoning models are compressed for edge deployment. The key loop-model issue is trajectory preservation: recursive models repeatedly update a latent state, so small compression errors can compound across reasoning cycles and break global task success even when local predictions still look acceptable.

## Added

### What Survives When You Compress a Recursive Reasoner for the Edge? (`2606.26488`)

This paper asks how recursive reasoning models behave under deployment-oriented compression. The authors test precision sweeps, pruning, distillation, linear attention, calibrated INT4, and edge memory constraints across structured tasks and recursive architectures.

Why it was added: the subject is explicitly recursive reasoning models that solve structured tasks by repeatedly updating a latent state. The paper studies the failure mode that matters for loop models: whether the recurrent reasoning trajectory survives compression. It belongs in analysis because the main contribution is diagnostic and deployment-oriented rather than a new base architecture.

Main reader takeaway: naive compression can preserve cell-level/local accuracy while collapsing puzzle-exact/global reasoning to zero. Carry-trajectory fidelity is introduced as a label-free signal for whether the compressed model still follows the full-precision reasoning path, and calibrated INT4 is reported to recover the failure without retraining.

## Checked but not added

- **CARVE: Content-Aware Recurrent with Value Efficiency for Chunk-Parallel Linear Attention** (`2606.27229`) improves recurrent sequence memory for delta-rule linear attention. It is useful adjacent work, but the recurrence is over token/state memory in a sequence model rather than reused depth within one forward reasoning process.
- **Erase-then-Delta Attention: Decoupling Erase and Write Addresses in Delta-Rule Linear Attention** (`2606.26560`) adds a learned erase direction to recurrent memory updates. It is again sequence-memory architecture, not a looped internal block or fixed-point/recurrent-depth model.
- **Error-Conditioned Neural Solvers** (`2606.27354`) iteratively corrects PDE predictions by feeding residual fields back into a learned solver. The learned update policy is interesting, but the abstract frames it as solver iteration for PDE surrogates rather than the loop-model families this repo tracks.
- **E-TTS: A New Embodied Test-Time Scaling Framework for Robotic Manipulation** (`2606.27268`) uses history-aware iterative refinement with vision-language verifiers. The loop is a modular test-time framework around candidate sampling and verification, not reuse of a learned internal module inside one model forward pass.
- **World Action Models Enable Continual Imitation Learning with Recurrent Generative Replays** (`2606.27374`) recursively queries a world-action model to synthesize replay trajectories. That is repeated external model querying for continual learning, outside the strict same-forward reused-module boundary.
