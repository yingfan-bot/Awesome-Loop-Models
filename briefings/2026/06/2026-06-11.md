# Daily Loop-Model Briefing - 2026-06-11

## Outcome

Four papers were added to the collection. The batch clears the recent June backlog around learned layer programs, latent VLA deliberation, hierarchical latent planning, and iterative geometric reconstruction. All four fit the strict collection scope because the useful computation happens through reused internal learned modules or repeated learned update operators, not through external agent loops or repeated prompting.

## Added to the collection

### Skip a Layer or Loop It? Learning Program-of-Layers in LLMs

PoLar treats pretrained LLM layers as reusable modules and learns input-specific execution programs over them. A generated program can skip layers or repeat them, so inference is no longer forced to follow the original fixed-depth, non-recurrent layer order. The paper reports that many inputs can keep or improve accuracy with shorter programs, and that some originally incorrect predictions can be corrected by alternate programs with fewer layers.

Why it was added: the abstract explicitly describes learned execution programs that dynamically skip or repeat pretrained layers inside inference. It belongs in designs because the main contribution is an architectural/inference mechanism for flexible layer reuse.

Main reader takeaway: fixed-depth LLM inference may be only one path through a richer set of latent computations already available in pretrained layers.

### MPCoT: Reward-Guided Multi-Path Latent Reasoning for Test-Time Scalable Vision-Language-Action

MPCoT brings latent deliberation into vision-language-action policies. Instead of emitting textual chain-of-thought, it initializes multiple latent hypotheses, refines them for K weight-tied steps, scores paths with reward-guided supervision, and softly aggregates them before decoding the action sequence. The paper reports gains on LIBERO and CALVIN, with ablations supporting the roles of depth, width, aggregation confidence, and reward-guided path supervision.

Why it was added: the central computation is a multi-path latent loop with explicit weight-tied refinement steps inside the policy computation. It belongs in applications because the looped mechanism is evaluated in long-horizon robotic control.

Main reader takeaway: latent loops can expose test-time compute controls for VLA policies without adding reasoning-token latency or changing the action interface.

### When to Re-Plan: Subgoal Persistence in Hierarchical Latent Reasoning

This paper studies a stability-adaptivity tradeoff in hierarchical latent reasoning. It extends HRM with a manager-worker interface where a slow high-level module emits a normalized directional subgoal that persists for P low-level update steps. Experiments on ARC and ConceptARC suggest that moderate persistence periods outperform both very frequent replanning and overly long commitments.

Why it was added: the mechanism is a hierarchical latent loop where a high-level recurrent signal persists across repeated low-level hidden-state updates. It belongs in designs because the paper isolates a concrete recurrent control knob for latent planning.

Main reader takeaway: in hierarchical loop models, the timing of high-level replanning can matter as much as the presence of a high-level signal.

### BA-T: An Iterative Transformer for Two-View Bundle Adjustment

BA-T adapts the structure of classical bundle adjustment into a compact Transformer decoder. Instead of stacking many independent decoder layers, it applies a single lightweight Transformer layer repeatedly in implicit token space, refining pose and reconstruction estimates from latent residuals across iterations. The paper reports progressive improvements over iterations and competitive accuracy with substantially fewer decoder parameters.

Why it was added: the abstract explicitly frames the model as an iterative Transformer with a repeatable learned layer for structured geometric refinement. It belongs in applications because the looped operator is used for two-view 3D reconstruction.

Main reader takeaway: looped Transformer updates are not only a language-model pattern; they can also encode classical iterative structure in geometric vision systems.

## Checked but not added

- **Dropout-GRPO: Variational Stochasticity for Continuous Latent Reasoning** (`2606.10184`) improves RL training for Coconut-style latent recurrence by using a shared dropout mask to create rollout diversity. It is useful context, but the main contribution is stochastic post-training rather than a new looped architecture or reused operator.
- **Why Limit the Residual Stream to Layers and Not Tokens? Persistent Memory for Continuous Latent Reasoning** (`2606.07720`) adds a gated persistent memory across latent reasoning passes. It is adjacent to the collection, but the abstract does not make same-forward reused learned-layer recurrence as explicit as the accepted papers.
- **Depth-Attention: Cross-Layer Value Mixing for Language Models** (`2606.05014`) improves cross-layer information flow and mentions gains for looped Transformers, but the central mechanism is cross-layer value retrieval rather than a reused learned block.
- **Geometric Latent Reasoning Induces Shorter Generations in LLMs** (`2606.02248`) uses a lightweight transition head for iterative embedding-space updates, but the abstract is less explicit about internal layer/block reuse than the accepted latent-loop papers.
- **Context-Driven Incremental Compression for Multi-Turn Dialogue Generation** (`2606.12411`) uses a retrieve-revise-write loop over dialogue memory across turns. That is an external conversation-memory process, not reused learned layers inside one forward computation.
- **Redesign Mixture-of-Experts Routers with Manifold Power Iteration** (`2606.12397`) uses a power-iteration-inspired router update principle, but it redesigns/trains MoE routers rather than making looped computation the deployed model mechanism.
- **Verifiable Environments Are LEGO Bricks: Recursive Composition for Reasoning Generalization** (`2606.12373`) recursively composes RL environments for training, so its recursion is outside the model.
- **Task-Aware Structured Memory for Dynamic Multi-modal In-Context Learning** (`2606.11853`) builds dynamic compressed memory for multimodal ICL, but the abstract does not identify a reused learned internal operator loop.
- **SVoT: State-aware Visualization-of-Thought for Spatial Reasoning via Reinforcement Learning** (`2606.11770`) trains interleaved textual and visual reasoning traces, but the repeated process is generated reasoning state rather than looped internal learned-module reuse.
