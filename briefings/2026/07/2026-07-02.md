# Daily Loop-Model Briefing - 2026-07-02

## Outcome

One new paper was added today: **G-RRM: Guiding Symbolic Solvers with Recurrent Reasoning Models** (`2607.02491`). It is not a general-purpose language-model architecture paper. Its value for this collection is narrower and cleaner: it tests whether a recurrent neural reasoner can provide useful guidance to exact symbolic solvers on constraint satisfaction problems.

## Added

### G-RRM: Guiding Symbolic Solvers with Recurrent Reasoning Models (`2607.02491`)

G-RRM uses symbol-equivariant Recurrent Reasoning Models (SE-RRMs) as neural solvers that generate full solution proposals, then passes those proposals as guidance to backtracking and SAT-style symbolic solvers. The paper reports that this works best when the symbolic solver has enough combinatorial search to benefit from hints and can dynamically overwrite bad branching choices.

Why it was added: SE-RRMs belong to the recurrent reasoning family this repository tracks: the learned solver repeatedly updates latent reasoning state rather than relying on an external prompt loop or plain autoregressive decoding. This paper is an applications-focused addition because the main result is about practical solver acceleration from that recurrent model, not a new base loop architecture.

Main reader takeaway: on 9x9 Sudoku, where the SE-RRM solves 91.1% of instances directly, G-RRM reduces median conflict counts to zero and reports a 33.3x median speedup for backtracking plus a 1.70x median speedup for Glucose 4.1. CaDiCaL does not benefit in the same way, which makes the paper useful as a boundary result: recurrent neural hints only translate into wall-clock gains when the downstream symbolic search procedure can exploit and repair them.

## Checked but not added

- **ReContext: Recursive Evidence Replay as LLM Harness for Long-Context Reasoning** (`2607.02509`) is a training-free evidence replay method around an existing LLM. It uses model-internal relevance signals, but the loop is a context-selection and replay harness rather than a reused learned internal operator inside one forward process.
- **A Hippocampus for Linear Attention: An Exact Memory for What the Recurrent State Forgets** (`2607.02303`) improves sequence-memory behavior for linear attention with a bounded exact KV cache. It is adjacent recurrent-state work, but the abstract is about token-prefix memory rather than recurrent depth or fixed-point loop-model computation.
- **Multimodal Knowledge Edit-Scoped Generalization for Online Recursive MLLM Editing** (`2607.01978`) is an online editing method with recursive updates and preconditioners. The recurrence is in the editing/update procedure, not the model's same-forward learned-module computation.
- **Closed-loop, agentic, and human-in-the-loop papers** from today's feed were left out unless they exposed an internal reused learned operator. Their loops are control, planning, verification, data, or deployment workflows rather than loop-model mechanisms.
