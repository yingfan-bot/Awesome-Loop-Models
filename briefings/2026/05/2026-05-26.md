## Added to the collection

### Simply Stabilizing the Loop via Fully Looped Transformer

This paper targets a core failure mode of looped transformers: increasing the number of loop iterations can destabilize training even though the architecture is attractive because it reuses the same Transformer blocks and allows test-time compute to be adjusted by changing the loop count. The authors trace the instability to gradient oscillation and residual explosion.

Their Fully Looped Transformer adds two parameter-free changes. Fully Looped Architecture distributes inter-loop signals across all layers to reduce residual explosion, and Attention Injection reuses existing attention blocks to suppress gradient oscillation. The reported result is stable training up to 12 loop iterations where baseline looped models collapse, and up to 13.2% average downstream-task improvement in easier regimes where baseline looped transformers still train.

Why it was added: the repeated reuse of Transformer blocks is explicit and central, and the contribution is a design paper about making that internal loop stable and useful under different test-time compute budgets.
