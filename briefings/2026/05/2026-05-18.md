# Daily Loop-Model Briefing — 2026-05-18

## Outcome

One high-confidence paper from the 2026-05-18 arXiv watch was added to the main collection: **Looped SSMs: Depth-Recurrence and Input Reshaping for Time Series Classification** (`2605.16048`). The inclusion evidence is explicit: the paper studies **depth-recurrence** in state-space models by reusing the same SSM block repeatedly across layers.

## Added to the collection

### Looped SSMs: Depth-Recurrence and Input Reshaping for Time Series Classification

State-space models are already recurrent along the sequence dimension, but this paper separates that from depth recurrence. It asks what happens when an SSM block with `k` parameters is iterated `L` times across depth instead of replacing it with `L` independently parameterized blocks. The authors test the idea across LRU, S5, LinOSS, and LrcSSM variants and combine it with input reshaping for low- and high-dimensional time-series inputs.

Why it is in scope: the loop is not a solver wrapper or repeated full-model call. The abstract explicitly says the model reuses the same block repeatedly across layers, making it a flat-loop architecture under the repository definition.

Main reader takeaway: despite having a strictly smaller hypothesis space than the untied `k · L`-parameter model, looped SSMs consistently match or outperform the untied baselines across six time-series classification benchmarks. The paper argues that depth parameter sharing acts as a beneficial inductive bias for optimization, and reports an additional 1–6% accuracy gain from input reshaping across all tested models.

## Checked but not added

### ATLAS: Agentic or Latent Visual Reasoning? One Word is Enough for Both

ATLAS introduces functional tokens that serve as both agentic operations and latent visual reasoning units while remaining ordinary vocabulary tokens trainable with standard SFT/RL pipelines.

Why it was not added: the abstract describes token-level latent/agentic operations, not a reused learned internal layer, block, module, or fixed-point operator applied multiple times within one model forward process.

### Context Pruning for Coding Agents via Multi-Rubric Latent Reasoning

LaMR decomposes code-context relevance into semantic-evidence and dependency-support rubrics, models them with dedicated CRFs, and fuses them with a gating network for coding-agent context pruning.

Why it was not added: this is structured context pruning for agent workflows. The repeated/structured computation is not a loop-model architecture under the repo’s same-forward reused-operator rule.

## Backlog note

It was submitted on 2026-05-08, so it is kept out of this daily intake and should be considered separately as a historical/backfill candidate if the maintainer wants a broader catch-up pass.
