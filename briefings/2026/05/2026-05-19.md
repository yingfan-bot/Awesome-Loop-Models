# Daily Loop-Model Briefing — 2026-05-19

## Outcome

Two high-confidence papers from the 2026-05-19 watch were added to the main collection:

1. **One Model, Two Roles: Emergent Specialization in a Shared Recurrent Transformer** (`2605.17811`)
2. **PERL: Parameter Efficient Reasoning in CLIP Latent Space** (`2605.18464`)

Both papers make the reused-operator evidence explicit in the abstract. The first studies a shared-weight recurrent Transformer reused for two update roles in Asymmetric Input Recurrence. The second applies a compact shared reasoning module recurrently across latent refinement steps for CLIP adaptation.

## Added to the collection

### One Model, Two Roles: Emergent Specialization in a Shared Recurrent Transformer

This paper asks whether a single shared recurrent Transformer can develop different internal functions without splitting into separate modules. It studies Asymmetric Input Recurrence (AIR), a two-state reasoning architecture where the same Transformer model is reused for both L- and H-state updates, with input injection distinguishing the update types.

Why it is in scope: the abstract explicitly says the same Transformer model is reused for both updates inside a recurrent reasoning architecture. The contribution is mechanistic analysis of what roles emerge across loop states, so it belongs in the analysis category.

Main reader takeaway: decoded rollouts, freezing experiments, ablations, and attention analysis suggest that state identity signals can induce stable role specialization inside shared-parameter recurrent Transformers. In the reported Sudoku-Extreme and Maze settings, H-like states behave more like committed proposal states, while L-like states retain more local uncertainty and shifting intermediate structure.

### PERL: Parameter Efficient Reasoning in CLIP Latent Space

PERL explores whether CLIP adaptation can come from iterative latent reasoning rather than simply adding more trainable parameters. It keeps CLIP frozen and adds a compact shared reasoning module that is applied recurrently across refinement steps, generating a latent reasoning token at each step and injecting it into an intermediate encoder layer.

Why it is in scope: the repeated computation is not an external agent loop or repeated full-model prompting. The abstract identifies a shared reasoning module reused across latent refinement steps within the adaptation model.

Main reader takeaway: across 15 base-to-novel, transfer, and out-of-distribution benchmarks, PERL reports the best parameter-performance trade-off among compared few-shot adaptation methods, using about 6K trainable parameters while keeping competitive transfer and novel-class accuracy.

## Checked but not added

### Leveraging Latent Visual Reasoning in Silence

This paper is relevant to latent visual reasoning, but its abstract focuses on whether latent tokens help training and post-training behavior. It does not state that a learned internal layer, block, or operator is reused repeatedly within one forward process, so it remains outside the strict paper-list scope.

### CineMatte: Background Matting for Virtual Production and Beyond

CineMatte uses a Siamese frozen DINOv3 ViT with shared weights to encode an input frame and background separately. That is weight sharing across two streams, not a looped/recurrent refinement process, so it was not added.

### Scale-Equivariant Generative Forecasting

The paper ties convolutional kernel weights across dilation levels for self-similar time series and is adjacent to weight-tied architecture design. It was kept as a watchlist item because the abstract does not clearly show a same learned operator repeatedly refining an internal state in the loop-model sense.
